# Style Guide

This repository should read like a practical reference, not like marketing copy.

## Preferred Style

- short paragraphs;
- concrete examples;
- clear tables for term mappings;
- explicit tradeoffs when translations vary.

## Avoid

- filler content;
- exaggerated claims;
- tool-specific hype without evidence;
- long copied passages from novels.

## Translation Terminology Notes

When there are multiple common renderings, prefer this structure:

1. list the recommended default;
2. mention notable alternatives;
3. explain when the default may not fit.
