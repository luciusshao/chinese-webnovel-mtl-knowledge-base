# Contributing

Thanks for contributing to this public knowledge base.

## What Good Contributions Look Like

- Clear, practical, and reusable advice
- Short examples that explain terminology or workflow
- Improvements to glossary consistency
- Prompt refinements with concrete before/after value
- Notes about legal and ethical boundaries for public sharing

## What Not To Contribute

- Full translated chapters from copyrighted novels
- Large excerpts that are not clearly fair-use quotations
- Pirated source material
- SEO filler or AI-generated fluff with no editorial review
- API keys, tokens, or private configuration

## Writing Style

- Prefer short sections and concrete examples
- Explain tradeoffs instead of claiming a single perfect translation
- Mark genre-specific preferences when relevant
- Use tables for glossary mappings when helpful
- Keep tone factual and readable

## Pull Request Scope

Small PRs are preferred. Good examples:

- add 10 common cultivation realm mappings;
- improve one prompt template;
- document one common MTL failure mode;
- clarify one policy page.

## Review Standard

PRs should be accepted when they are:

- useful to readers or tool builders;
- legally safe to publish;
- easy to maintain;
- not tied to reposting copyrighted text.
